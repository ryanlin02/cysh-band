# 網站聯絡與帳號維護指引

本文件記錄嘉義高中管樂隊暨校友管樂團網站的公開聯絡入口與共同帳號原則。目標是讓網站、社群與資料保存不綁在單一個人的私人帳號上。

## 1. 目前公開聯絡入口

- 網站頁面：`contact.html`（「聯絡與資料更正」）。
- 對外按鈕目前寄至組織 Gmail：`cysh.band.official@gmail.com`。
- 網頁不直接顯示此 Gmail 地址；寄件者點選「寄送電子郵件」才會開啟自己的郵件軟體。
- 適用事項：內容勘誤、校友資料補充、歷史素材、照片／文字授權、要求更正或下架、網站技術意見。

處理來信時，不要把寄件者的電話、地址、身分證字號、私人聯絡方式、未公開名冊或群組截圖直接貼回網站資料夾。只有確認可公開的資料，才可依既有公開資料流程整理。

## 2. 組織 Gmail 管理

帳號：`cysh.band.official@gmail.com`

用途：網站聯絡收件、Google Search Console、Google Drive／表單、YouTube 與未來其他官方服務的主要管理身分。

請立刻完成：

1. 啟用兩步驟驗證。
2. 加入至少兩位可信任管理者可協助的復原方式（例如復原信箱與備用碼），但不要把密碼貼在群組或網站 repo。
3. 將備用碼與帳號交接紀錄保存在受控的團隊文件或密碼管理工具，並註明至少兩位保管人。
4. 社群平台優先使用「邀請管理者／角色權限」，不要多人共用密碼。

## 3. 日後升級為 contact@cysh.band（非必要）

現階段可免費使用組織 Gmail，無須 Google Workspace。日後若希望往來寄件者也看見 `@cysh.band`，可設定 `contact@cysh.band`：

1. 先確認目前網域的 MX／轉寄設定沒有正在使用的地址。
2. 在 DNS 或郵件轉寄服務建立 `contact@cysh.band`，收件轉寄到組織 Gmail。
3. 實際寄一封外部測試信，確認可收到。
4. 再把 `contact.html` 的 mailto 目的地改為 `contact@cysh.band`。

僅有轉寄服務時，回覆通常仍會顯示 Gmail 地址。只有未來確實需要以 `@cysh.band` 收發信、多位成員共用信箱或管理更多雲端服務時，再評估付費的 Google Workspace 或其他 SMTP 郵件服務。

## 4. 售票資料的更新規則

在正式 OPENTIX 購票頁、最低票價與開賣時間都已確認前，不要在首頁 MusicEvent 的 JSON-LD 加入 `offers`，也不要把 OPENTIX 首頁當作購票連結。

售票正式公開後，才一次更新：

- `index.html` 的活動 CTA 與 MusicEvent `offers`。
- `concerts.html` 的活動 CTA。
- `data/concerts.js` 第 41 屆的 `ticket`。
- 相關最新消息、`網站修改歷程紀錄.md`。

完成後執行 `node scripts/generate-concert-pages.js`、`node scripts/check-site.js` 與 `git diff --check`。
