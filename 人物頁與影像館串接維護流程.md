# 人物頁與影像館串接維護流程

文件建立時間：2026-07-11  
適用範圍：`people/{id}.html` 個人介紹頁、`data/people-profiles.js`、`photos/` 影像館前端、影像館 R2 人物索引。

## 一、資料角色

1. `data/people-profiles.js`
   - 官網人物個人頁的主要 metadata 來源。
   - 個人頁大頭照 `photo` 是正式人物照片來源。
   - 若個人頁大頭照為本人提供或經人工挑選，優先於影像館人臉縮圖。

2. `photos/profile-links.js`
   - 由 `scripts/generate-gallery-profile-links.js` 產生。
   - 提供影像館使用的「人物編號／站內個人頁／正式大頭照」對照。
   - 不可手動修改；若內容不一致，重新產生。

3. 影像館 `people.json`
   - 存放於 Cloudflare R2：`https://img.cysh.band/data/people.json`。
   - 管理影像館中的人物辨識結果、照片數量與影像館人物縮圖。
   - 官網串接時主要使用 `num` 對應，不使用顯示姓名作為唯一判斷。

## 二、前台連結規則

1. 個人介紹頁 → 影像館
   - 個人頁由 `scripts/generate-people-pages.js` 產生。
   - 只要人物有 `num`，頁面會出現「影像館中的照片」區塊。
   - 連結格式固定為：

```text
../photos/#/person-num/{編號}
```

2. 影像館 → 個人介紹頁
   - `photos/app.js` 會讀取 `photos/profile-links.js`。
   - 若影像館人物的 `num` 對應到官網個人頁，人物照片頁標題旁會顯示「閱讀人物介紹」。
   - 影像館人物列表、人物照片頁與燈箱人物標籤會優先使用個人頁正式大頭照；若正式照片是 `blank.webp` 或沒有對應個人頁，才使用影像館人物縮圖。

3. 無編號人物
   - 無 `num` 的人物無法用穩定編號自動串接。
   - 若未來需要串接，應先補上可信的公開編號；不得只靠姓名自動對應。

## 三、更新流程

### 3.1 新增或修改個人介紹頁

1. 更新 `content/people/{id}.html` 與 `data/people-profiles.js`。
2. 若大頭照有變更，更新 `assets/img/members/{id}.webp`，並確認 `data/alumni.js` 的 `photo` 與個人頁一致。
3. 執行：

```bash
node scripts/generate-people-pages.js
node scripts/generate-gallery-profile-links.js
node scripts/generate-people-index.js
node scripts/check-site.js
node scripts/link-people-names.js --check
git diff --check
```

4. 更新 `網站修改歷程紀錄.md`。

### 3.2 影像館新增照片或重新產生人物辨識資料

1. 在影像館資料產生流程中確認 `people.json` 的人物有正確 `num`。
2. 若只是新增照片、更新人物照片數量或調整影像館 avatar，且官網人物資料未變更，通常不需要修改本 repo。
3. 若新增的人物已經有或即將有官網個人頁，需回到本 repo 更新 `data/people-profiles.js`，並重新產生 `photos/profile-links.js`。
4. 上線後抽查：
   - `photos/#/person-num/{編號}` 能進到正確人物照片頁。
   - 影像館人物頁姓名旁有「閱讀人物介紹」。
   - 個人頁底部「影像館中的照片」能連回影像館。

### 3.3 更換正式大頭照

1. 以個人介紹頁正式照片為主。
2. 更新 `assets/img/members/{id}.webp` 與 `data/people-profiles.js`。
3. 執行 `node scripts/generate-gallery-profile-links.js`，讓影像館同步使用新正式照片。
4. 執行 `node scripts/check-site.js`，確認 `data/alumni.js`、人物誌卡片與影像館對照表沒有照片不同步。

## 四、避免錯誤的原則

- 不用姓名自動串接人物頁與影像館，因為可能有同名、別名、舞台名或姓名格式差異。
- 不手改 `photos/profile-links.js`；此檔只接受產生器輸出。
- 不把影像館辨識縮圖反向覆蓋個人頁正式大頭照。
- 個人頁是正式人物介紹來源；影像館是照片出現紀錄來源。
- 若影像館辨識後續發現少數錯誤，應修正 R2 的照片人物索引，不需要改官網個人頁資料。

## 五、目前串接狀態

截至 2026-07-11 建立本流程時，37 個個人介紹頁中，有 33 個可在影像館公開人物資料中以 `num` 對應。尚未在影像館 `people.json` 找到對應的人物，個人頁仍會以穩定的 `person-num` 路由連結；影像館若日後補上同編號人物，即可自動導向正確人物頁。
